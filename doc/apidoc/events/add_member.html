POST /api/v1/events/:id/members
Associate an user or team to the event's team

Also see events#assignable_members.

Supported Formats

json, xml

Errors

400 Bad Request. he server cannot or will not process the request due to something that is perceived to be a client error.
401 Unauthorized access
404 The requested resource was not found
406 The server cannot return data in the requested format
422 Unprocessable Entity: The change could not be processed because of errors on the data
500 Server crashed for some reason. Possible because of missing required params or wrong parameters

Examples

Adding an user to the event members
POST: /api/v1/events/8383/members.json
DATA:
{
  'memberable_id': 1,
  'memberable_type': 'user'
}

RESPONSE:
{
  'success': true,
  'info': "Member successfully added to event",
  'data': {}
}

Adding a team to the event members
POST: /api/v1/events/8383/members.json
DATA:
{
  'memberable_id': 1,
  'memberable_type': 'team'
}

RESPONSE:
{
  'success': true,
  'info': "Member successfully added to event",
  'data': {}
}

Params

Param name Description
memberable_id
required

The ID of team/user to be added as a member


Value:

Must be a number.
memberable_type
required

The type of element to be added as a member


Value:

Must be one of: user, team.
