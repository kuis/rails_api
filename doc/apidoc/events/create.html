POST /api/v1/events
Create a new event

Supported Formats

json, xml

Errors

400 Bad Request. he server cannot or will not process the request due to something that is perceived to be a client error.
401 Unauthorized access
404 The requested resource was not found
406 The server cannot return data in the requested format
422 Unprocessable Entity: The change could not be processed because of errors on the data
500 Server crashed for some reason. Possible because of missing required params or wrong parameters

Event's Place ID. This can be either an existing place id that is already registered on the application, or the combination of the place reference + place id returned by Google's places API. (See: developers.google.com/places/documentation/details). Those two values must be concatenated by '||' in the form of '<reference>||<place_id>'. If using the results from the API's call: Venues Search, you should use the value for the id attribute
