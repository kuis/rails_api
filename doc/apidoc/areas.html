GET /api/v1/areas
Get a list of areas

Returns a list of areas sorted by name. Only those areas that are accessible for the user will be returned.

Each area item have the following attributes:

  • id: the area's ID

  • name: the area's name

Supported Formats

json, xml

Errors

400 Bad Request. he server cannot or will not process the request due to something that is perceived to be a client error.
404 Missing
401 Unauthorized access
500 Server crashed for some reason

Examples

GET /api/v1/areas.json
[
  {
      "id": "1",
      "name": "Albany"
  },
  {
      "id": "2",
      "name": "Arizona"
  },
  {
      "id": "3",
      "name": "Asbury Park"
  },
  ...
]
>>>

GET /api/v1/areas/:id/cities
Get a list of cities for an Area

Also see areas#index.

Returns a list of the valid cities for an area.

Each city item have the following attributes:

  • id: the city's ID

  • name: the city's name

  • state: the city's state

  • country: the city's country code

Supported Formats

json, xml

Errors

400 Bad Request. he server cannot or will not process the request due to something that is perceived to be a client error.
404 Missing
401 Unauthorized access
500 Server crashed for some reason

Examples

GET /api/v1/areas/1/cities.json
[
    {
        "id": 5212,
        "name": "Albany",
        "state": "New York",
        "country": "US"
    }
]

Params

Param name Description
id
required

The area's ID.


Value:

Must be a number.
