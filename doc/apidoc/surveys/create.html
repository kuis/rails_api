POST /api/v1/events/:event_id/surveys
Create a new survey for a event

Also see surveys#brands.

An answer have to have a one of the following convinations:

  • kpi_id and answer

  • brand_id, question_id and answer

Supported Formats

json, xml

Errors

404 Missing
401 Unauthorized access
403 Event does allows surveys as per campaign settings
500 Server crashed for some reason

Params

Param name Description
auth_token
required

User's authorization token returned by login method


Value: Must be String
company_id
required

One of the allowed company ids returned by the “User companies” API method


Value: Must be a number.
survey
required
Value: Must be a Hash
survey[surveys_answers_attributes]
required

This should receive a list of result, each one with the following structure:

  • kpi_id:, The kpi_id of this answer. “6” for gender, “7” for age or “8” for ethnicity. (if this is an answer that is related to a kpi. Must be one of: [6,7,8]

  • brand_id: The ID of the brand for this answer, if this is an answer that is related to a brand. This should be the ID of one of the brands assigned to the event. See surveys#brands for more detail

  • question_id: The number of the question for this answer. Must be one of [1,2,3,4]

  • answer: The value for this answer. Depending of what is this for, the value can be one of the following:

    For the Age KPI:

    • 1: < 12

    • 2: 12 – 17

    • 387: 18 – 20

    • 3: 21 – 24

    • 4: 25 – 34

    • 5: 35 – 44

    • 6: 45 – 54

    • 7: 55 – 64

    • 8: 65+

    For the ethnicity KPI:

    • 11: Asian

    • 12: Black / African American

    • 13: Hispanic / Latino

    • 14: Native American

    • 15: White

    For the gender KPI:

    • 9: Female

    • 10: Male

    For Question #1: Can be any of: [“purchased”, “aware”, “unaware”]

    For Question #2: Should be an integer/float value

    For Question #3: Can be any of: [1, 2, 3, 4, 5]

    For Question #4: Can be any of: [1, 2, 3, 4, 5]


Value: Must be a list of answers [id, answer].
